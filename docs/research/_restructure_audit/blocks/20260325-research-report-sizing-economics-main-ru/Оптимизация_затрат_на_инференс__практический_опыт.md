## Оптимизация затрат на инференс (практический опыт)

Связка с моделью затрат: рычаги ниже напрямую бьют по веткам **Инференс** и **Эксплуатация** в дереве факторов стоимости и снижают требования к GPU при тех же SLO — их стоит явно закладывать в чувствительность сценарного сайзинга.

**Источник:** Канал @Redmadnews (red_mad_robot) [[источник]](https://t.me/Redmadnews)

### Снижение вычислительных затрат

**Google Think@n (Deep-Thinking Ratio):**
- Метрика DTR оценивает активность мышления на уровне слоёв
- Снижение вычислительных затрат ~2x при сопоставимой точности

**Oppo AI SMTL:**
- Параллельное решение подзадач вместо последовательных рассуждений
- Сокращение шагов инференса на 70.7%

**Moonshot Attention Residuals:**
- Избирательное использование представлений из прошлых слоёв
- Снижение вычислений в 1.25x

### Оптимизация памяти агентов

**Accenture Memex(RL):**
- Индексированная память вместо раздутого контекста
- Пиковое потребление токенов -50%
- Успешность задач: 24.2% → 85.6%

**Databricks KARL:**
- Корпоративный поиск с RL
- 33% дешевле, 47% быстрее чем Claude 4.6 / GPT-5.2
