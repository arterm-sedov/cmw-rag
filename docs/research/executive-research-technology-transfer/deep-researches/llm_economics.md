Putting the Model Into Practice  \n\nA practical per‑token cost calculator can be expressed as:\n\n```\nCost_per_token = \n   (GPU_hour_rate / tokens_per_hour)               // compute\n + (GPU_TDP/1000 * 1/throughput * PUE * $/kWh)    // power\n + (License_rate_per_hour / tokens_per_hour)      // software\n + (Storage_per_token + Egress_per_token)         // data\n + (Personnel_annual / Annual_token_volume)       // ops\n```\n\n- **GPU_hour_rate**: on‑demand, spot, or amortised purchase cost.  \n- **tokens_per_hour**: measured throughput for the target model/precision (e.g., 3,500 t/s on H100).  \n- **GPU_TDP**: 300 W (A100) or 700 W (H100/H200).  \n- **PUE**: 1.58 average, 1.10 best‑in‑class (source 9).  \n- **$/kWh**: region‑specific electricity price.  \n- **License_rate_per_hour**: $1 / GPU‑hr (cloud) or $4,500 / yr (on‑prem).  \n- **Storage/Egress**: from provider pricing tables.  \n- **Personnel_annual**: sum of salaries for engineers, SREs, security staff allocated to the LLM service.\n\nPlugging the numbers for a given deployment yields a transparent, token‑level cost that can be compared across on‑prem and cloud options, and fed back into budgeting, chargeback, or product pricing decisions.\n\n--- \n\n**Bottom line:**  \n- **On‑prem** offers lower marginal cost per token only when utilisation is very high (>80 %) and electricity is cheap;
