## 1. Минимизация данных и приватность

### Подтверждённые практики

**OpenTelemetry GenAI (официальная рекомендация):**
- **По умолчанию НЕ записывать** системные инструкции, полные сообщения и вывод модели в атрибуты спанов
- Для зрелого продакшна рекомендуется паттерн **внешнего хранилища контента** с **ссылками** в телеметрии
- Раздельный контроль доступа (SRE vs разработка)

**Связь с 152-FЗ:**
- Маскирование ПДн в логах — обязательно
- Сроки хранения журналов — согласовать с ДПО
- Матрица доступа — разделение ролей
- Исключение экспорта в недоверенные SaaS без ДПО

### Паттерн каскадной минимизации

```
Запрос пользователя
    ↓
[1] Детерминированные правила (телефоны, email, реквизиты)
    ↓
[2] Контекстные фильтры + малые модели распознавания сущностей
    ↓
[3] Основная генерация с семантическими плейсхолдерами
    ↓
[4] Обратное отображение для пользователя (безопасно)
```

## 2. Инцидент-менеджмент и мониторинг

### Обязательные сигналы для GenAI

| Категория | Метрики |
|-----------|---------|
| **Токены** | Входные/выходные, кэширование |
| **Задержки** | TTFT, время на токен, общее время отклика |
| **Ошибки** | Rate limits, timeout, API errors |
| **Агентский цикл** | Глубина, число итераций, повторы вызовов |
| **Политики** | Guardrails срабатывания, эскалации, отказы по политике |

### Runbook разбора инцидентов

**По trace_id:**
1. Извлечь трассу из backend (Phoenix/Langfuse/SigNoz)
2. Идентифицировать этап сбоя (embeddings → retrieval → generation → tools)
3. Проверить политики (guardrails, allowlist)
4. Проанализировать входные данные на предмет инъекции
5. Оценить использование токенов и ресурсов

**SLO для GenAI:**
- Доступность API: 99,9%
- Время отклика P95: < 5 сек (для синхронных запросов)
- Время первого токена P95: < 2 сек
- Ошибки LLM: < 1%

### Политика сэмплинга

| Тип трафика | Сэмплинг | Хранение |
|-------------|----------|----------|
| Ошибки | 100% | 90 дней |
| Canary-деплой | 100% | 30 дней |
| Продакшн | 10-50% (настраивается) | 7 дней |
| Debug-режим | 100% | 24 часа |

## 3. Российская специфика

### Регуляторные требования

| Требование | Статус | Источник |
|------------|--------|----------|
| Локализация ПДн в России | ✅ Обязательно | 152-FЗ, ст. 18 |
| Регистрация оператора ПДн | ✅ Обязательно | Роскомнадзор |
| Уведомление о начале обработки ПДн | ✅ Обязательно | Роскомнадзор |
| Анонимизация при передаче | ✅ Обязательно | 152-FЗ, 23-ФЗ |
| Согласие на обработку ПДн | ✅ Обязательно | 152-FЗ, ст. 9 |

### Для государственного сектора и КИИ

- **Резидентный контур** — обязательно
- **Сертификация** — FSTEC и ФСБ выдают сертификаты для AI-систем на критических объектах
- **Ограничения** — запрет на использование иностранных AI-решений без согласования
