## 1. Обзор

В данном документе представлен подробный анализ аппаратных требований (сайзинга), капитальных затрат (CapEx) и операционных затрат (OpEx) для развертывания экосистемы ИИ CMW с фокусом на российский рынок и локальные облачные провайдеры. Анализ основан на фактических измерениях VRAM из `cmw-mosec`, спецификациях vLLM из `cmw-vllm` и тарифах российских облачных платформ (Cloud.ru, Yandex Cloud, SberCloud).

**Ключевые выводы:**
- **Мелкомасштабное развертывание:** ~5 ГБ VRAM (модели 0.6B) подходит для RTX 4090 или облачного A10G.
- **Среднемасштабное развертывание:** ~18 ГБ VRAM (модели 4B) требует высокопроизводительный GPU (RTX 6000 Ada, A6000).
- **Крупномасштабное развертывание:** >48 ГБ VRAM (модели 8B) требует дата-центральные GPU (A100, H100).

## 8. Стратегии оптимизации затрат

1.  **Выбор моделей:** Используйте модели 0.6B для приложений с чувствительностью к стоимости; резервируйте 4B+ для высокоточных задач.
2.  **Смешанная точность:** Используйте fp16/bf16 для уменьшения VRAM на 50% по сравнению с fp32.
3.  **Обслуживание нескольких моделей:** MOSEC позволяет развертывать несколько моделей на одном сервере, снижая потребности в оборудовании.
4.  **Облачные Spot-инстансы:** Используйте AWS Spot или GCP Preemptible для экономии 60-90% на некритичных рабочих нагрузках.
5.  **Автоматическое масштабирование:** Масштабируйте облачные инстансы в зависимости от спроса для минимизации простоев.

## 9. Заключение

Экосистема ИИ CMW предлагает гибкие варианты сайзинга, подходящие для различных масштабов клиентов:
- **Мелкомасштабные** развертывания возможны на потребительских GPU (RTX 4090).
- **Среднемасштабные** требуют профессиональные GPU (RTX 6000, A10G).
- **Крупномасштабные** требуют дата-центральные GPU (A100, H100).

**Торговля CapEx и OpEx:**
- **Локальное развертывание:** Более высокие первоначальные затраты, более низкая долгосрочная TCO для устойчивых рабочих нагрузок.
- **Облако:** Нулевые первоначальные затраты, более высокая долгосрочная TCO, идеально для переменных рабочих нагрузок.

**Рекомендация:** Начните с облачного развертывания для концепции-доказательства, затем перейдите на локальное развертывание для продакшена, если рабочая нагрузка стабильна и предсказуема.
