# GPU TCO / break-even table validation (report-pack)

**Дата:** 2026-04-03

## Ситуация

В методологическом отчёте стояла сводная таблица «TCO Cloud vs On-Prem» для **8×H100** с фиксированными суммами и **70 %** «экономии» без воспроизводимой связи с прайсами комплекта.

## Проблема

- Суммы **не сходились** с открытыми тарифами **Cloud.ru** и вилками **Selectel** из того же отчёта по сайзингу.
- Смешивались разные основания: **3 года аренды**, **годовая** величина и **цена покупки** одной карты в одной сетке без формул.
- Колонка «амортизация» для on-prem при фактически приведённых цифрах вводила в заблуждение (показаны были уровни, близкие к **CapEx только GPU**, а не полный TCO).

## Решение

- **Канонический** количественный блок перенесён в _«Отчёт. Сайзинг и экономика»_ → раздел **TCO GPU: облако РФ против закупки** (якорь `#sizing_gpu_cloud_onprem_tco_breakeven`).
- Пересчёт выполнен от **публичного** прайса Cloud.ru (**7×H100 NVLink**, масштабирование на **8 GPU**) и **вилки** Selectel; закупка — из существующей матрицы цен GPU.
- Пороги утилизации **40 % / 60–70 %** согласованы с обзорной публикацией (Ivchenko, 2026, Zenodo) и помечены как **не** автоматическая норма для РФ без пересчёта прайса.
- В методологии якорь **`#method_infrastructure_breakeven`** сохранён; таблица заменена на **ссылку** на сайзинг (избежание дублирования противоречивых цифр).
