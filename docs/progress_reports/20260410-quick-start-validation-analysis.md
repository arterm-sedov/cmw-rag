## 0. Ключевой вывод

**Quick Start сравнивает три модели развёртывания для ОДИНАКОВОЙнагрузки — это НЕ то же самое, что сравнение разных размеров развёртывания в sizing report.**

- Sizing report показывает TCO для разных размеров: мелкое/среднее/крупное

## 1. Найденные расхождения в цифрах TCO

### 1.1 Внутренняянеисправность в Quick Start

**Таблица в Quick Start (строки 189-195):**

| Статья | On-prem (RTX PRO 6000) |
|--------|------------------------|
| CapEx | 1,200,000 руб. |
| OpEx (инфраструктура) | 300,000 руб./год |
| OpEx (поддержка и LLMOps) | 1,200,000 руб./год |
| **TCO (1 год)** | **2,700,000 руб.** |
| **TCO (3 года)** | **5,100,000 руб.** |

**Проверка расчёта:**
- Год 1: CapEx (1.2M) + OpEx инфра (0.3M) + OpEx LLMOps (1.2M) = 2.7M ✓
- Год 2: OpEx инфра (0.3M) + OpEx LLMOps (1.2M) = 1.5M
- Год 3: OpEx инфра (0.3M) + OpEx LLMOps (1.2M) = 1.5M
- **TCO 3 года = 1.2M + 4.5M = 5.7M** ≠ **5.1M** (в документе)

**Расхождение: 600,000 руб.** ← Ошибка в арифметике

## 2. Причины расхождений

### 2.1 Разные задачи сравнения

| Аспект | Sizing Report | Quick Start |
|--------|---------------|-------------|
| Сравнивает | Размеры развёртывания | Модели владения для одной нагрузки |
| Цель | Выбор размера под нагрузку | Выбор: облако vs аренда vs on-prem |
| Переменная | CapEx/OpEx по размеру | CapEx/OpEx по модели владения |

### 2.2 Разные составы OpEx

**Sizing report OpEx (425k/год) включает:**
- Обслуживание железа
- Электричество
- Базовая поддержка

**Quick Start OpEx (1.5M/год) включает:**
- 300k/год — инфраструктура (энергия, ЦОД)
- 1.2M/год — поддержка и LLMOps (команда)

LLMOps team costs = ~1M/год, что объясняет разницу.

### 2.3 Несоответствие между таблицей и детальным разделом в Sizing

**Старый sizing report:**
- Таблица (строки 1140-1147): CapEx 850k, OpEx 425k/yr
- Детально (строки 1169-1171): CapEx 1.2-2.0M, OpEx 425k/yr

**Restructured sizing report:**
- Таблица (строки 766-773): CapEx 850k, OpEx 425k/yr
- Детально (строки 801-823): CapEx 1.2-2.0M, OpEx 425k/yr

**Проблема:** Два источника CapEx для «Среднее» — 850k и 1.2-2.0M.

## 4. Предлагаемая гармонизация

### Вариант A: Скорректировать Quick Start под sizing report (рекомендуется)

**Обоснование:** Quick Start сравнивает модели владения, sizing показывает размеры. Нужно согласовать профиля.

**Предложение для Quick Start, таблицаTCO (строки 189-195):**

| Статья затрат | Облачный API | Аренда GPU (1dedic 2×RTX 4090 48 ГБ) | On-prem (RTX PRO 6000) |
|:--- | :--- | :--- | :--- |
| **CapEx (оборудование)** | 0 | 0 | **1 200 000 руб.** |
| **OpEx (инфраструктура)** | 2 400 000 руб./год | 1 200 000 руб./год | **300 000 руб./год** |
| **OpEx (поддержка и LLMOps)** | 300 000 руб./год | 600 000 руб./год | **425 000 руб./год** |
| **Итого TCO (1 год)** | **2 700 000 руб.** | **1 800 000 руб.** | **1 925 000 руб.** |
| **Итого TCO (3 года)** | **8 100 000 руб.** | **5 400 000 руб.** | **2 775 000 руб.** |

**Расчёт для On-prem:**
- CapEx: 1.2M (в рамках «Среднее предприятие» sizing)
- OpEx инфра: 300k/год (энергия, ЦОД)
- OpEx LLMOps: 425k/год (согласовано с sizing)
- TCO 1 год = 1.2 + 0.3 + 0.425 = 1.925M
- TCO 3 года = 1.2 + 3×(0.3 + 0.425) = 1.2 + 2.175 = 3.375M

**Но это нарушает логику сравнения моделей:** Cloud API становится дороже on-prem уже на 1-м году.

### Вариант B: Оставить Quick Start как есть, добавить пояснение

**Обоснование:** Quick Start моделирует «продвинутый» сценарий с полной командой LLMOps, что отличается от типового «Среднего» в sizing.

**Редакция для Quick Start:**

```markdown
**Примечание:** Цифры TCO в данном разделе учитывают полный состав команды LLMOps 
(аналитик, разработчик, DevOps, эксперт предметной области) и отличаются от 
типового «Среднего» развёртывания в _«[Сайзинг и экономика](./20260325-research-report-sizing-economics-main-ru_restructured.md#sizing_tco_3_year_comparison)»_.
```
