## Цель

Обогатить отчёты **реальной инженерной базой Comindware**: агентный контур как проверенная платформа. Текущие отчёты описывают отраслевые лучшие практики — они валидны и сохраняются. Comindware добавляется как **конкретная реализация** поверх абстрактных описаний, показать **что уже работает**, а не только «что планируется».

**Принцип:** обогащение, не переписывание. Отраслевые подходы остаются — Comindware добавляется как доказательство реализации. Бизнес-ценность превалирует над техническими деталями.

## Что изменилось в понимании

После глубокого сканирования всех репозиториев, презентации ассистента аналитика и отраслевого исследования:

### Агентный контур Comindware Platform — экосистема компонентов

| Компонент | Роль в платформе |
|-----------|------------------|
| **RAG-движок** | RAG-контур на любых базах знаний, MCP-серверы, структурированный анализ запросов, конвейеры индексации и глубоких исследований, оценка качества |
| **Ассистент аналитика Comindware** | Агентный слой для прямого взаимодействия с Comindware Platform: инструменты для управления сущностями, мультипровайдер LLM, сессионная изоляция |
| **Сервер инференса MOSEC** | Унифицированный сервер специализированных моделей (эмбеддинг, реранк, гард) с YAML-конфигурацией и CLI, управляемый нашим фреймворком |
| **Сервер инференса vLLM** | Промышленный инференс LLM с OpenAI-совместимым API, управляемый нашим фреймворком |
| **Сервер инференса Infinity** | Сервер эмбеддингов и реранкеров на базе infinity-emb, управляемый нашим фреймворком |
| **Анонимайзер** | Внутренний фреймворк извлечения и маскирования метаданных из потоков сообщений LLM |

### Отраслевой контекст (подтверждено веб-исследованием)

- **Bain & Company, апрель 2026:** «Три слоя агентной ИИ-платформы» — подтверждает архитектуру: слой данных → слой агентов → слой оркестрации. Comindware покрывает все три.
- **MCP (Model Context Protocol, протокол контекста модели):** 28% компаний Fortune 500 уже развернули MCP-серверы (начало 2026). Comindware уже имеет MCP-совместимые эндпоинты.
- **Schema-Guided Reasoning (SGR, структурированное рассуждение по схеме):** 5–10% улучшение точности по сравнению с неструктурированными промптами; воспроизводимое рассуждение, аудит каждого шага. Comindware использует SGR в нескольких точках конвейеров: для анализа запросов, критики ответов агентов и детерминированного управления любым мышлением.
- **LangGraph:** актуальная версия — мультиагентная оркестрация. Comindware использует агентную архитектуру на базе LangChain с middleware.

- **Целевая аудитория:** аналитики (настройка приложений, аудит, реверс-инжиниринг ТЗ) + конечные пользователи (поиск информации, создание записей, отчёты)

  1. Замкнутый контур клиента: Platform + ассистент + LLM локально, без интернета
