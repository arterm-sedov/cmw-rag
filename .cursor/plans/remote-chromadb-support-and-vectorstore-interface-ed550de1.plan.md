# Remote ChromaDB Support and VectorStore Interface Implementation

## Overview

Implement remote ChromaDB connection support via `HttpClient` while maintaining backward compatibility with local `PersistentClient` mode. Simultaneously implement the VectorStore Protocol interface from Phase 2 to enable pluggable vector store backends.

## Implementation Plan

### Phase 1: Create VectorStore Protocol Interface

#### 1.1 Create Storage Interfaces (`rag_engine/storage/interfaces.py` - NEW)

Create Protocol-based interface matching Phase 2 specification:

```python
"""Protocol-based interfaces for vector storage backends."""
from __future__ import annotations
from typing import Any, Dict, List, Optional, Protocol

class VectorStore(Protocol):
    """Protocol for vector storage backends."""
    
    def add(
        self,
        texts: List[str],
        metadatas: List[Dict[str, Any]],
        ids: Optional[List[str]] = None,
        embeddings: Optional[List[List[float]]] = None,
    ) -> None:
        """Add documents to the vector store."""
        ...
    
    def similarity_search(
        self, query_embedding: List[float], k: int = 5
    ) -> List[Any]:
        """Search for similar documents."""
        ...
    
    def get_any_doc_meta(
        self, where: Dict[str, Any]
    ) -> Optional[Dict[str, Any]]:
        """Return metadata of any one document matching a metadata filter."""
        ...
    
    def delete_where(self, where: Dict[str, Any]) -> None:
        """Delete all records matching a metadata filter."""
        ...
```

### Phase 2: Update Settings Configuration

#### 2.1 Update Settings (`rag_engine/config/settings.py`)

Add remote connection configuration fields:

```python
# ChromaDB
chromadb_persist_dir: str = ""  # Required for local mode
chromadb_collection: str
# Remote connection (optional - if set, uses remote mode)
chromadb_host: str = ""  # e.g., "192.168.1.100" or "chromadb.example.com"
chromadb_port: int = 8000
chromadb_ssl: bool = False
```

**Configuration Logic:**

- If `chromadb_host` is set (non-empty): Use `HttpClient` (remote mode)
- If `chromadb_host` is empty: Use `PersistentClient` (local mode), requires `chromadb_persist_dir`

#### 2.2 Update `.env-example`

Add remote connection options with clear documentation:

```env
# ChromaDB
# Local mode (default): specify CHROMADB_PERSIST_DIR
CHROMADB_PERSIST_DIR=./data/chromadb_data
CHROMADB_COLLECTION=mkdocs_kb

# Remote mode: specify CHROMADB_HOST to use remote ChromaDB server
# Uncomment and configure the following for remote connection:
# CHROMADB_HOST=192.168.1.100
# CHROMADB_PORT=8000
# CHROMADB_SSL=false
```

### Phase 3: Update ChromaStore Implementation

#### 3.1 Update ChromaStore (`rag_engine/storage/vector_store.py`)

Modify to support both local and remote modes:

1. **Update `__init__` method** to accept optional remote connection parameters
2. **Update `collection` property** to lazily initialize either `PersistentClient` or `HttpClient` based on configuration
3. **Add Protocol compliance comment** indicating `ChromaStore` implements `VectorStore` protocol
4. **Maintain backward compatibility** with existing `persist_dir` + `collection_name` signature

Key changes:

- Union type for `_client`: `Optional[Union[chromadb.PersistentClient, chromadb.HttpClient]]`
- Configuration validation: Ensure either `persist_dir` or `host` is provided
- Lazy initialization in `collection` property with mode detection

#### 3.2 Create Factory Function (`rag_engine/storage/vector_store.py`)

Add convenience function to create ChromaStore from settings:

```python
def create_chroma_store_from_settings(settings) -> ChromaStore:
    """Create ChromaStore instance from settings.
    
    Automatically selects local or remote mode based on chromadb_host.
    Returns ChromaStore instance configured according to settings.
    """
```

### Phase 4: Update Application and Scripts

#### 4.1 Update Application Entry Point (`rag_engine/api/app.py`)

Replace direct ChromaStore instantiation with factory function:

```python
from rag_engine.storage.vector_store import create_chroma_store_from_settings

# Replace:
# vector_store = ChromaStore(
#     persist_dir=settings.chromadb_persist_dir,
#     collection_name=settings.chromadb_collection,
```

**Approach**: Use factory function where settings are available, otherwise support both modes via command-line arguments or environment variables.

````markdown
## ChromaDB Configuration

### Local Mode (Default)

The RAG engine uses local file-based ChromaDB by default:

```env
CHROMADB_PERSIST_DIR=./data/chromadb_data
CHROMADB_COLLECTION=mkdocs_kb
````

### Remote Mode

Connect to a ChromaDB server running on another machine:

**1. Start ChromaDB Server on Remote Machine:**

```bash
# Using Docker (recommended)
docker run -p 8000:8000 chromadb/chroma

# Or using ChromaDB CLI
chroma run --host 0.0.0.0 --port 8000
```

**2. Configure Client in .env:**

```env
CHROMADB_HOST=192.168.1.100
CHROMADB_PORT=8000
CHROMADB_SSL=false
CHROMADB_COLLECTION=mkdocs_kb
```

**Note:** When `CHROMADB_HOST` is set, the engine automatically uses remote mode. `CHROMADB_PERSIST_DIR` is not required for remote connections.

### VectorStore Interface

The engine uses a Protocol-based `VectorStore` interface (`rag_engine/storage/interfaces.py`), enabling pluggable backends. `ChromaStore` implements this interface and supports both local and remote connections.
